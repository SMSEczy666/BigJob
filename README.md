# 大作业

## 一些共识



* 题型1 数字，数组

* 题型2 线性表，字符串

* 题型3 查找，排序

* 题型4 树，图



* 维度1 提交次数
* 维度2 debug时间（最后一次提交时间 - 第一次提交时间）单位秒
* 维度3 有效行数（纯代码的行数，除去了注释、空行）
* 维度4 mi指数 （0-100的数，越大越好）[参考：如何科学评估代码复杂度？](https://juejin.im/post/5cdd7bdb5188256903366410)

## 一阶段filter

+ 若某题debug时间超过24小时 or 有效行数为0行 or MI指数是100，则不分析该题

+ 若用户某题型做不满5题，则不分析该用户
